§ I Why this page exists

The churn model I built at Hancock Whitney is under NDA — the dataset, the feature library, the exact thresholds, the lift numbers. None of that can live on a portfolio. So this page runs the same pipeline against a public dataset, step for step: drop IDs, encode categoricals, stratified split, three models side by side, tree SHAP for explainability, a group-wise fairness audit, and a cost-weighted threshold.

The Kaggle Churn Modelling set is smaller and softer than real bank data, so the ceiling here is around 85–87% test accuracy — not the number the production model hits. Expect a plain dashboard, not a magic trick. What this page is really showing you is the shape of a responsible modeling workflow: measure three models, explain the winner, check it for disparate impact, then tune the threshold against real dollars.

Everything below is reproducible. Clone the repo, run the script, regenerate the JSON, and these charts redraw themselves.

      § VII · FIG. G.4 Feature importance — mean |SHAP|, XGBoost

      Tree SHAP decomposes each XGBoost prediction into per-feature contributions and averages their absolute values across a 500-row test sample. Read it as: which features move the score the most, regardless of direction.

      § X Methodology & Colophon
      Dataset

      Kaggle · shrutimechlearn/churn-modelling ↗ — 10,000 synthetic bank customers across France, Germany, Spain.

      Pipeline script

      notebooks/churn_model.py ↗ — drop IDs, label-encode Gender, one-hot Geography, stratified 80/20 split, StandardScaler on numeric features, three models, SHAP on XGB, fairness slice on the best model, threshold sweep 0.05 → 0.95.

      Production model

      The Hancock Whitney churn model — its feature library, decision thresholds, and lift numbers — is under NDA. This page mirrors the workflow, not the proprietary model.

      Reproducibility

      random_state=42 everywhere. Last regenerated —. Clone the repo, run python notebooks/churn_model.py, regenerate all nine JSON files.
