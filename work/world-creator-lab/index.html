FIG. 10 An AI as a 3D world-builder

Type a place.
Get a world.

A prompt becomes a self-contained 3D scene, generated live by an AI model and rendered in your browser. No game engine, no asset library, no download. The model writes the geometry, the lighting, and the camera, all in one HTML file.

Five worlds load instantly. Click a card and the scene swaps in place. Below that, a custom prompt input — type your own and the AI writes a brand-new world for you. Live generation takes about a minute and costs a fraction of a dollar.

§ I The model is the renderer

A frontier language model is good enough at code that you can ask it to be the renderer. Skip the engine, the asset pipeline, the texture downloads. One prompt, one HTML file, one scene.

Peter Gostev's project, claude-opus-47-context-worlds, pre-baked three of these scenes (Voxel Rome, Stonehenge, Golden Gate) to compare Claude Opus 4.7 at 400k vs 1M context. The observation that motivated this lab is the same as his: a single prompt to a strong model now produces a renderable world.

This lab takes that idea and makes it interactive. You type the place; the page sends it to Claude on OpenRouter; the response streams back as an HTML scene, sandboxed in an iframe, rendered to your screen. Each generation costs roughly a dime. The site caches by prompt for a week, so the second visitor who asks for "Machu Picchu at sunset" gets the first visitor's render instantly.

§ III How it works
A · Prompt

A short scene description

The user types a place or mood. The page tightens it with a system prompt that tells the model exactly what to output: one complete HTML document, Three.js inline, no external assets, muted palette, subtle animation, 60fps target.

B · Generation

Streamed from Claude on OpenRouter

A Cloudflare Pages Function streams the response from deepseek/deepseek-v4-pro. About 50 KB of HTML and JavaScript over 30 seconds. The Function caches the result by prompt hash for a week so repeat queries are free.

C · Render

Sandboxed iframe, no escape

The HTML is dropped into an iframe with sandbox="allow-scripts" and no same-origin permission. Scripts run, the scene paints, but the iframe cannot read this page, set cookies, or fetch arbitrary URLs.

§ IV · FIG. 10.2 What stops this from running up a bill

Live AI generation is genuinely expensive. Public LLM endpoints without controls get drained in hours. The defenses here are layered, cheap rejections first, model calls last.

01

Hard input cap

800 characters at the client and the server. Anything longer is rejected before any model call.

02

Per-IP rate limit

Three generations per IP per hour, six per day. IPs are hashed before storage.

03

Topic guard

Keyword pre-check rejects prompts that aren't about places, scenes, or 3D worlds. No LLM cost on rejection.

04

Hard output cap

16,000 tokens. Long enough for a rich scene, short enough to bound the cost per call.

05

Seven-day cache

Identical prompts return cached results for a week. Repeat traffic costs nothing.

06

$5 daily spend cap

Hard limit. The endpoint returns a polite refusal once the cap trips, no matter who's asking.

§ V Methodology, Credits & Colophon
Pipeline

functions/api/world-creator.js ↗ is the Cloudflare Pages Function that streams the generation. It reads the user prompt, applies the safety layer, and calls deepseek/deepseek-v4-pro via OpenRouter. The response is streamed as Server-Sent Events and rendered in the iframe on this page.

The seed scene

The Machu Picchu scene that loads first was hand-written in Three.js by me, not generated. It establishes the visual target the model is aiming at: low-poly stylization, atmospheric lighting, slow camera orbit. Every generated scene gets compared to this seed implicitly.

Credit

This lab is directly inspired by Peter Gostev's claude-opus-47-context-worlds ↗ (MIT). Peter's project pre-bakes three scenes to compare Claude Opus 4.7 at two context windows. This lab takes that "AI as renderer" thesis and makes it live and interactive. Thank you to Peter for shipping the original and licensing it openly.

Honest caveats

Generation quality varies wildly with prompt specificity and the model's mood. About 1 in 10 generations fails to render cleanly (broken JavaScript, missing geometry). The cache hides successful runs from later visitors but does nothing for the first attempt. The seed scene is the floor; generated scenes sometimes exceed it, sometimes don't.
